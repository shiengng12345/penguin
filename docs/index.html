One workspace for gRPC-Web, gRPC, and SDK testing. 把 gRPC-Web、gRPC、SDK 测试放进同一个工作区。

Penguin is built for QA and developers who need to install packages, browse services and methods, edit JSON bodies, send requests, inspect responses, and reuse requests quickly. Penguin 适合 QA 和开发者：安装 package、浏览 service/method、编辑 JSON body、发送请求、查看响应、快速复用请求。

Core capabilities 核心能力

What the app gives you day to day. 这个 app 日常能帮你做什么。

Package-driven browser 包驱动浏览器

Install packages and explore services and methods from the sidebar. 安装 packages 后，直接在侧边栏浏览 services 和 methods。

Structured request editor 结构化请求编辑

Edit headers and JSON body, format/reset content, and auto-fill defaults from definitions. 编辑 headers 和 JSON body，可格式化、重置，并从定义自动填默认值。

Response inspection 响应查看

Inspect status, headers, and body with handling for large payloads. 查看 status、headers、body，并对大响应做更稳的展示。

Environment switching 环境切换

Use per-protocol environments with {{URL}} interpolation for repeated testing. 每个协议都有独立环境，支持 {{URL}} 变量插值。

Reusable workflow 可复用工作流

Save requests, reopen from history, copy as cURL, and keep common test flows close. 保存请求、从历史重新打开、复制成 cURL，把常用测试流程留在手边。

Built-in tools 内建工具

Network check, cURL import, request docs, proto viewer, updater, and more. 网络检查、cURL 导入、请求文档、proto 查看、更新检查等都内建。

Daily workflow 日常流程

The typical path through the app. 你平时在 app 里大概会这样走。

1

Choose a protocol and package 选协议和 package

Start in gRPC-Web, gRPC, or SDK, then install or select a package. 先决定走 gRPC-Web、gRPC 还是 SDK，然后安装或选择 package。

2

Pick a service and method 选 service 和 method

Use the sidebar tree or search to load the method into the current tab. 用侧边栏或搜索，把 method 载入当前 tab。

3

Prepare request data 准备请求资料

Set URL, switch environment, adjust headers, and edit the JSON body. 设置 URL、切环境、调整 headers、编辑 JSON body。

4

Send and inspect 发送并查看

Send the request, inspect the response panel, and compare headers/body. 发送请求后，在响应面板检查结果、headers 和 body。

5

Save or branch the flow 保存或分支测试流程

Save the request, open a new tab, or reuse a history item for the next test. 保存请求、开新 tab，或从 history 继续往下测。

Protocol modes 协议模式

Three ways to call APIs, one interface. 三种调用方式，共用一套界面。

gRPC-Web

ConnectRPC transport with a Rust proxy for browser-friendly gRPC-Web traffic. ConnectRPC 搭配 Rust proxy，适合处理浏览器风格的 gRPC-Web。

gRPC

Native gRPC executed through a Node sidecar using grpc-js and proto-loader. 通过 Node sidecar 跑 grpc-js 和 proto-loader，直接打原生 gRPC。

SDK

SDK method execution through a Node runtime that imports the package bundle directly. 通过 Node runtime 直接载入 SDK bundle，然后执行方法。
